# Telegram Digest - инструкция по настройке

Система ежедневной сводки из телеграм-чатов поставщиков WB.

## Как это работает

1. **18:00** - GitHub Actions запускает скрипт, который через Telethon вытягивает сообщения из 5 чатов за последние 24 часа и коммитит JSON-файл в репозиторий
2. **19:00** - Cowork scheduled task забирает этот файл, анализирует и выдаёт сводку

## Шаг 1: Получить Session String (5 минут)

У тебя уже есть api_id и api_hash. Осталось получить session string.

На своём маке в терминале:

```bash
pip3 install telethon
python3 setup_session.py
```

Скрипт попросит:
- api_id → введи `26572095`
- api_hash → введи свой hash
- номер телефона → введи свой номер
- код подтверждения → придёт в Telegram

После этого скрипт выведет длинную строку - это Session String. Скопируй её.

## Шаг 2: Создать GitHub-репозиторий (5 минут)

1. Зайди на github.com → New repository
2. Название: `telegram-digest` (приватный!)
3. Загрузи в него файлы:
   - `telegram_exporter.py`
   - `requirements.txt`
   - `.github/workflows/telegram_export.yml`

## Шаг 3: Добавить секреты в GitHub (3 минуты)

В репозитории: Settings → Secrets and variables → Actions → New repository secret

Добавь 4 секрета:

| Название | Значение |
|----------|----------|
| `TELEGRAM_API_ID` | `26572095` |
| `TELEGRAM_API_HASH` | твой api_hash |
| `TELEGRAM_SESSION_STRING` | строка из шага 1 |
| `TELEGRAM_CHATS` | JSON-массив чатов (см. ниже) |

### Формат TELEGRAM_CHATS

Нужны username или ID пяти чатов. Примеры:

```json
["@wb_chat_1", "@wb_chat_2", "@wb_sellers", -1001234567890, -1009876543210]
```

Как узнать ID чата: перешли любое сообщение из чата боту @userinfobot или @getidsbot.

## Шаг 4: Тестовый запуск (2 минуты)

1. В репозитории: Actions → Telegram Daily Export → Run workflow
2. Подожди ~30 секунд
3. Проверь что в папке `data/` появился файл `export_YYYY-MM-DD.json`
4. Если ошибка - проверь логи в Actions

## Шаг 5: Настроить Cowork scheduled task

Scheduled task уже создан и запускается каждый день в 19:00.

**Важно:** после создания репозитория нужно обновить URL в задаче, подставив твой GitHub username. Скажи мне username - я обновлю.

Также рекомендую нажать "Run now" на задаче один раз, чтобы одобрить доступы к инструментам.

## Шаг 6: Сделать репо доступным для Cowork

Два варианта:

**Вариант A (проще):** Сделать репо публичным - тогда Cowork сможет читать файлы по прямой ссылке. Данные в JSON - это просто сообщения из чатов, ничего секретного.

**Вариант B (безопаснее):** Оставить репо приватным, создать Personal Access Token (GitHub → Settings → Developer settings → Tokens) и добавить его в настройки задачи.

## Что получишь

Каждый день в ~19:00 в Cowork будет появляться сводка:
- Главные темы дня
- Важные новости и изменения на WB
- Проблемы и жалобы поставщиков
- Полезные советы и лайфхаки
- Интересные цифры
- Рекомендации что стоит прочитать целиком
